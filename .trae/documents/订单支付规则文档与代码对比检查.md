# 订单支付规则文档与代码对比检查计划

## 检查范围
- 文档：`/workspace/doc/V0.5/订单支付规则.md`
- 代码：`order_product.html`、`order_movie.html`、`order_cake.html`
- 选择页：`points_batch_select.html`、`coupon_select.html`、`movie_card_select.html`

---

## 发现的问题

### 🔴 严重 Bug

#### 1. order_product.html `goToCouponSelect` 未设置 `return_page`
- **位置**: `order_product.html` 第 1240-1245 行
- **问题**: `goToCouponSelect()` 函数没有设置 `localStorage.setItem('return_page', 'order_product.html')`，而 `coupon_select.html` 依赖 `return_page` 来决定返回地址
- **影响**: 如果用户之前访问过蛋糕订单并设置了 `return_page = 'order_cake.html'`，那么从商品订单进入优惠券选择页后，确认选择会跳转到蛋糕订单页而非商品订单页
- **对比**: `goToBatchSelect()` 正确设置了 `return_page`，`order_cake.html` 的 `goToCouponSelect()` 也正确设置了
- **修复**: 在 `goToCouponSelect()` 中添加 `localStorage.setItem('return_page', 'order_product.html')`

### 🟡 中等 Bug

#### 2. movie_card_select.html 不使用 `return_page` 动态返回
- **位置**: `movie_card_select.html` 第 229 行
- **问题**: `confirmSelect()` 和 `goBack()` 硬编码 `location.href = 'order_movie.html'`，而非读取 `localStorage.getItem('return_page')`
- **文档描述**: "各选择页面通过 localStorage 的 return_page 字段动态返回对应的订单页面"
- **影响**: 虽然目前电影次卡选择页仅从电影票订单进入，硬编码不会导致错误，但与文档规范和其他选择页（points_batch_select、coupon_select）的实现不一致
- **修复**: 改为 `location.href = localStorage.getItem('return_page') || 'order_movie.html'`

#### 3. order_movie.html `goToMovieCardSelect` 未设置 `return_page`
- **位置**: `order_movie.html` 第 1050-1054 行
- **问题**: 跳转到电影次卡选择页时未设置 `return_page`
- **影响**: 由于 movie_card_select.html 硬编码了返回地址，目前不会出错，但与文档规范不一致
- **修复**: 在 `goToMovieCardSelect()` 中添加 `localStorage.setItem('return_page', 'order_movie.html')`

#### 4. 电影次卡全额抵扣时，支付方式区域显示空白
- **位置**: `order_movie.html` 第 981-985 行 + 第 1042-1045 行
- **问题**: 当 `movieActive = true` 且 `cardValue >= total` 时，`currentMode = 'movie'`，不渲染任何支付方式选项，但支付方式区域（含"支付方式"标题）仍然显示（因为 `movieActive = true`），导致出现一个只有标题没有内容的空白区域
- **文档描述**: "仅电影次卡 | 面值 >= 总额 | 无选项（直接全额抵扣）"
- **修复**: 当 `currentMode === 'movie'` 时，隐藏支付方式区域

### 🟢 轻微问题

#### 5. order_product.html combo-input-section 初始状态有 `show` class
- **位置**: `order_product.html` 第 843 行
- **问题**: HTML 中 `<div class="combo-input-section show" id="comboInput">` 默认带有 `show` class，导致页面加载时组合支付输入框短暂可见，直到 JS 初始化后才会被正确隐藏
- **修复**: 移除 HTML 中的 `show` class，改为 `<div class="combo-input-section" id="comboInput">`

#### 6. order_cake.html pay-channel-section 初始状态缺少 `show` class
- **位置**: `order_cake.html` 第 793 行
- **问题**: HTML 中 `<div class="pay-channel-section" id="payChannel">` 没有 `show` class，但默认模式为 `cash`（需要显示现金渠道），导致页面加载时现金支付渠道短暂不可见
- **修复**: 添加 `show` class，改为 `<div class="pay-channel-section show" id="payChannel">`

#### 7. order_movie.html 跳转积分批次选择页时未设置 `order_products`
- **位置**: `order_movie.html` 第 1056-1062 行
- **问题**: `goToBatchSelect()` 没有设置 `localStorage.setItem('order_products', ...)`，而 `points_batch_select.html` 会读取 `order_products` 来显示商品摘要和进行积分匹配
- **影响**: 积分批次选择页的"当前订单商品"区域为空，但由于电影票订单只支持通兑积分（匹配所有商品），功能上不受影响
- **修复**: 在 `goToBatchSelect()` 中构造电影票商品数据并设置到 `order_products`

## 代码与文档一致的部分 ✅

1. **商品订单**：积分批次支持全部5种、优惠券支持、支付方式动态渲染、计算顺序（优惠券→积分→现金）均正确
2. **电影票订单**：电影次卡与积分互斥、次卡计算规则（min(pricePerUse, 票价) × min(remaining, 张数)）、支付方式动态渲染均正确
3. **蛋糕订单**：仅支持通兑积分、优惠券支持、地址只读、数量固定显示、配送方式固定均正确
4. **积分换算**：1积分 = 1分 = ¥0.01，代码中正确使用分为单位（乘以100）
5. **localStorage 键值**：order_type、nav_source、selected_batch_id、selected_coupon_id、selected_movie_card_id、order_products 均正确使用
6. **收银台弹窗**：2步流程（确认+密码）、纯积分支付跳过收银台均正确实现
7. **蛋糕订单收银台**：同时显示积分抵扣和优惠券抵扣明细，正确实现
