# 福利商城 — 产品原型项目

## 定位

PM 产品文档项目，产出微信小程序积分商城（江苏银行"积分"体系）的原型 HTML、业务逻辑清单、PRD 文档。
项目本质是**聚合平台**，不是自营商城 — 融合了自营商品、外部 H5（蛋糕叔叔）、第三方供应商（电影票）。

## 技术约束

- 纯 HTML/CSS/JS，无框架无构建，必须兼容 file:// 协议
- 设计系统在 css/common.css（375px 画框、微信小程序 UI 规范）
- Mock 认证：js/auth.js（测试账号 13800001111/2222/3333，支持微信绑定流程模拟）
- 登录守卫：js/auth-guard.js（页面引入后未登录自动跳转 login.html）
- Mock 数据：js/movie_pass_cards.js（电影次卡/点卡演示数据）

## 核心业务复杂度

### 三种订单类型，各自独立支付逻辑

| 类型 | 页面 | 支付方式 |
|------|------|----------|
| 商品订单 | order_product.html | 积分+优惠券+现金组合，计算顺序：券→积分→现金补差 |
| 蛋糕订单 | order_cake.html | web-view 嵌入蛋糕叔叔 H5，JS Bridge 拦截下单走我方支付 |
| 电影票订单 | order_movie.html | 应付金额 + 次卡/点卡抵扣 + 补款，三选一互斥 |

### 组合支付计算规则

1. 优惠券抵扣（满门槛校验）
2. 积分抵扣：min(余额, 券扣减后剩余金额)，按商品价格权重分配
3. 现金补差：商品总额 - 优惠券 - 积分
4. 详细规则见 doc/V0.5/订单支付规则.md

### 电影卡券互斥

- 每单三选一：不使用 / 仅一张点卡 / 仅一张次卡
- 点卡：储值型，余额扣减，归零失效
- 次卡：计次型，有剩余次数和单次抵扣上限
- 明确不支持：次卡+点卡叠加、单订单多张次卡

### 外部合作方集成

- **蛋糕叔叔**：web-view 嵌入 H5，联合登录授权(JWT) → 商品浏览 → JS Bridge 拦截下单 → 我方统一支付 → 状态回调同步
- **电影票供应商**：选座锁座 → 卡券抵扣 → 支付 → 出票二维码
