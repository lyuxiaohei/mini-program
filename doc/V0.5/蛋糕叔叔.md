# 蛋糕叔叔 · 联调流程图

```mermaid
flowchart TD
    %% 颜色定义（部分环境不支持时可删除 class 段，图仍可显示）
    classDef userSide fill:#e1f5fe,stroke:#01579b,stroke-width:1px
    classDef ourSystem fill:#e8eaf6,stroke:#3f51b5,stroke-width:1px
    classDef partner fill:#fff3e0,stroke:#ef6c00,stroke-width:1px
    classDef payment fill:#ffebee,stroke:#c62828,stroke-width:1px

    subgraph P1["阶段一 联合登录授权"]
        A["用户打开小程序<br/>微信授权，获取 openid"] -->|"数据传递"| B["我方 Auth 服务<br/>生成联合 Token JWT"]
        B -->|"数据传递"| C["蛋糕叔叔 Auth 接口<br/>验证 Token，返回 session_key"]
        C -.->|"绑定关系持久化 openid 与 uid"| A
    end

    subgraph P2["阶段二 H5 嵌入与浏览"]
        D["小程序 web-view<br/>加载带 Token 的 URL"] -->|"URL 参数携带加密 Token"| E["蛋糕叔叔 H5 页面<br/>商品浏览、加购、配置"]
    end

    subgraph P3["阶段三 统一支付"]
        E -->|"触发下单"| F["H5 发起下单请求<br/>postMessage 通知小程序"]
        F <-->|"JS Bridge 拦截"| G["我方订单服务<br/>创建聚合订单，生成单号"]
        G --> H["微信支付收银台<br/>wx.requestPayment 调起"]
    end

    subgraph CB["支付异步回调"]
        H -.->|"用户确认"| I["支付成功页<br/>展示结果并跳转订单页"]
        J["微信支付回调 异步<br/>验签与幂等处理"] -.-> I
        J --> K["蛋糕叔叔履约系统<br/>接单、生产、配送"]
        L["我方支付状态更新<br/>更新订单并通知对方履约"] -->|"推送支付成功通知"| K
        J -.-> L
    end

    subgraph P4["阶段四 订单状态同步与展示"]
        K -.->|"状态变更回调"| M["我方订单状态机<br/>待支付至制作中、配送中、完成"]
        M --> N["小程序订单列表<br/>轮询与订阅消息推送"]
        M --> O["后台管理系统<br/>搜索、筛选、售后、导出"]
        M --> P["财务对账与数据报表<br/>日结对账与异常单预警"]
    end

    class A,D,I,N userSide
    class B,G,L,M,O,P ourSystem
    class C,E,F,K partner
    class H,J payment
```

## 若仍无法显示

1. **必须用围栏**：上图已用 ` ```mermaid ` … ` ``` ` 包裹；纯 `graph TD` 无围栏时多数 Markdown 不会当 Mermaid 渲染。
2. **避免特殊符号**：原图边标签中的 `↔`、`&`、`+` 等已改为普通中文，减少解析器报错。
3. **子图标题**：改用 `subgraph id["标题"]` 双引号包裹，且标题内不用英文冒号 `:`，避免与子图语法冲突。
4. **节点内换行**：原 `<br/>` 在部分 Mermaid 版本中对中文不稳定，已改为空格分段；若需换行可改回 `"第一行<br/>第二行"` 并自行在本地预览验证。
