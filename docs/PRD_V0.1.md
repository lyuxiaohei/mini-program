### 商品数据来源说明

> **全量商品数据：** 所有页面中商品卡片展示的商品信息（名称、价格、图片、规格、销量等）均来源于品牌商城后台的商品管理列表中的商品数据。后台运营人员通过商品管理模块进行商品的增删改查、上下架等操作，前端页面实时读取后台维护的商品数据。
>
> **组件商品数据：** 首页等页面中组件/活动模块展示的商品数据来源于魔方配置。运营通过魔方配置工具选择和排列展示的商品，组件商品数据 ≤ 全量商品数据（即魔方中只能配置后台已有的商品）。

### 统计指标定义

| 指标 | 定义 | 计算规则 |
|------|------|----------|
| 销量 | 按SKU维度统计已产生订单的商品数量 | SKU销量 = 该SKU已完成订单中该SKU的商品件数之和 |
| 好评率 | SKU维度的好评占比 | 好评率 =（该SKU的五星好评数量 ÷ 该SKU的全部评价数量）× 100% |
| 折扣 | 由后端计算并返回前端展示 | 折扣 = 现价 ÷ 原价 × 10，保留一位小数（如3.9折）。后端计算，前端直接展示 |

### 2.2 微信登录（login.html 弹窗）

> **微信用户信息获取方式说明：** `wx.getUserProfile` 已废弃（返回匿名数据）。当前微信小程序获取用户头像和昵称需使用以下组件方案：
> - **头像：** `<button open-type="chooseAvatar">` 组件，用户主动选择头像，通过 `bindchooseavatar` 事件获取临时文件路径
> - **昵称：** `<input type="nickname">` 组件，用户输入时微信键盘上方显示昵称快捷填充

#### 操作流程

1. 点击"微信登录"按钮 → 弹出手机号绑定弹窗（输入手机号+确认按钮）
2. 输入手机号 → 确认：
   - 未注册手机号 → 弹出"设置个人信息"页面：
     - 头像选择区：使用 `<button open-type="chooseAvatar">` 让用户选择头像，或使用默认头像
     - 昵称输入框：使用 `<input type="nickname">` 让用户输入/选择微信昵称，或系统生成随机昵称
     - 确认 → 自动注册并登录 → 首页
   - 已注册未绑微信 → "绑定成功，正在登录" → 首页
   - 已注册已绑微信 → "该手机号已绑定其他微信" → 弹窗不关闭
3. 中途关闭绑定弹窗 → 清理临时会话，提示"已取消微信登录"

#### 业务规则

| 规则编号 | 规则描述 |
|----------|----------|
| RULE-WX-001 | 点击弹窗外蒙层可关闭授权弹窗和绑定弹窗 |
| RULE-WX-002 | 绑定弹窗中手机号校验同手机号登录（非空、格式） |
| RULE-WX-003 | 绑定冲突检测：若手机号已绑定其他微信，返回 `already_bound` 错误 |
| RULE-WX-004 | 未注册手机号通过微信登录时，让用户选择是否使用微信信息 |

### 3.4 收货地址（address.html）

#### 页面结构

地址列表 + 底部新增按钮 + 弹窗表单（新增/编辑共用，顶部含粘贴识别区域）。

#### 粘贴识别

新增/编辑弹窗顶部提供"粘贴地址识别"文本区域。用户粘贴一段完整地址文本后，系统自动解析并填充下方表单字段。

**识别规则：**
1. 提取手机号：匹配 `1[3-9]` 开头的11位数字
2. 提取地区：匹配省/市/区/县等行政区划关键词（如"江苏省南京市鼓楼区"）
3. 提取收件人：剩余文本中长度≤5且不含长数字的短文本
4. 提取详细地址：包含"路""街""号""栋""楼""室"等地址关键词的文本段

**支持格式示例：**
- `张三 13812345678 江苏省南京市鼓楼区汉中路100号`
- `张三，13812345678，江苏省南京市鼓楼区汉中路100号18楼`
- `李四 13900001111 上海市浦东新区陆家嘴环路1000号恒生银行大厦`

**触发时机：**
- 粘贴（paste）事件触发后自动解析并填充
- 手动输入超过10个字符后延迟800ms自动解析

2. 拒绝 → 弹窗关闭，无变化
3. 允许 → 检查 `Auth.isWeChatBoundToOther(phone)`

1. 点击微信绑定（已绑定状态）→ confirm"确定要解绑微信吗？"

## 五、我的收藏

### 5.0 收藏维度与入口

**收藏维度：基于SKU（最小销售单元）。**

用户收藏的是某个商品的具体规格组合（如"白色 500ml"），而非SPU（商品）。同一SPU下不同SKU视为不同收藏项。

**收藏操作流程（从商品详情页）：**

1. 用户在商品详情页点击心形收藏图标
2. 若商品有多个SKU规格，弹出规格选择弹窗（同加入购物车的规格选择器）
3. 用户选择具体规格（颜色/尺寸等）→ 点击"确认收藏"
4. 心形图标变红，提示"已收藏"
5. 若商品仅有一个SKU，点击心形直接收藏，无需弹窗选择
6. 重复收藏同一SKU不新增记录，心形保持红色状态

## 八、数据模型

### 8.1 用户数据（localStorage: `syd_accounts` + `syd_current`）

```javascript
{
  phone: String,           // 11位手机号
  name: String,            // 昵称（随机生成或用户修改）
  avatar: String,          // 头像URL（默认SVG或微信头像）
  gender: String,          // 'male' | 'female' | 'secret'
  birthday: String,        // 'YYYY-MM-DD'，默认 '1970-01-01'
  email: String,           // 邮箱（选填）
  wechatBound: Boolean,    // 是否绑定微信
  wechatNickname: String   // 微信昵称
}
```

### 7.2 Auth API

| 方法 | 说明 |
|------|------|
| `Auth.getCurrentUser()` | 获取当前登录用户 |
| `Auth.isLoggedIn()` | 是否已登录（非临时会话且有手机号） |
| `Auth.loginWithPhone(phone)` | 手机号登录，未注册自动注册 |
| `Auth.loginWithWeChat()` | 创建微信临时会话 |
| `Auth.bindPhoneToWeChat(phone)` | 微信登录绑定手机号，冲突返回 `{error:'already_bound'}` |
| `Auth.applyWeChatInfo()` | 使用微信头像和昵称 |
| `Auth.applyRandomInfo()` | 使用默认头像和随机昵称 |
| `Auth.updateCurrentUser(updates)` | 更新当前用户信息（同步到账号列表） |
| `Auth.bindWeChat()` | 绑定微信到当前账号 |
| `Auth.unbindWeChat()` | 解绑微信 |
| `Auth.isWeChatBoundToOther(phone)` | 检查微信是否已绑定其他账号 |
| `Auth.clearTempSession()` | 清理未完成的微信登录临时会话 |
| `Auth.logout()` | 退出登录，清除登录态 |
| `Auth.resetAccounts()` | 重置所有账号数据 |
| `Auth.maskPhone(phone)` | 手机号脱敏（138****8888） |

### 9.3 登录拦截

受保护页面加载时通过 `auth-guard.js` 检查登录状态：
```javascript
if (!Auth.isLoggedIn()) {
  Auth.clearTempSession();
  location.replace('login.html');
}
```

受保护页面列表：profile、settings、profile_edit、change_phone、order、order_list、order_detail、payment、address、favorites、coupon、wallet、points_detail
