# 订单支付规则

## 商品订单（order_product.html）

### 金额明细入口（始终显示，可叠加）

| 入口 | 说明 | 选择页面 |
|------|------|----------|
| 积分抵扣 | 支持全部5种积分批次（指定商品/特殊商品/品牌商品/分类商品/通兑） | points_batch_select.html |
| 优惠券 | 满门槛或无门槛抵扣 | coupon_select.html |

### 支付方式（根据金额明细选择动态渲染）

| 金额明细选择 | 支付方式 | 说明 |
|---|---|---|
| 都不选 | 现金支付 | 支付方式区隐藏 |
| 仅积分 | 积分支付 / 组合支付 | 组合支付=积分+现金 |
| 仅优惠券 | 现金支付 | 优惠券折扣后现金支付剩余 |
| 积分+优惠券 | 积分支付 | 优惠券先扣，积分抵扣剩余 |

### 计算顺序

1. 优惠券抵扣：`discount`（满门槛校验）
2. 积分抵扣：`min(余额, 优惠券扣减后的剩余金额)`，按商品价格权重分配到各商品
3. 现金补差：`商品总额 - 优惠券 - 积分`

## 电影票订单（order_movie.html）

### 金额明细入口（始终显示，互斥）

| 入口 | 说明 | 选择页面 |
|------|------|----------|
| 电影次卡 | 多不退少补：`min(pricePerUse, 票价)` × `min(remaining, 张数)` | movie_card_select.html |
| 积分抵扣 | 仅支持通兑商品积分，其他批次置灰不可选 | points_batch_select.html |

> 电影次卡与积分互斥：选择其中一个会自动取消另一个。

### 支付方式（根据金额明细选择动态渲染）

| 金额明细选择 | 支付方式 | 说明 |
|---|---|---|
| 都不选 | 现金支付 | 支付方式区隐藏 |
| 仅电影次卡 | 电影次卡支付 / 组合支付 | 组合支付=次卡+现金（次卡面值不够票价时） |
| 仅积分 | 积分支付 / 组合支付 | 组合支付=积分+现金（积分不够时） |

### 计算顺序

1. 电影次卡抵扣：`min(remaining, 张数)` 次，每次 `min(pricePerUse, 票价)`
2. 积分抵扣：`min(余额, 剩余金额)`
3. 现金补差：`票款总额 - 次卡抵扣 - 积分抵扣`

### 电影次卡规则示例

票价 ¥50/张：

| 次卡 | pricePerUse | 买1张票 | 买2张票 |
|------|-------------|---------|---------|
| 单人观影次卡（3次） | ¥35 | 抵¥35，补¥15现金 | 抵¥70（2次×¥35），补¥30现金 |
| VIP观影次卡（1次） | ¥60 | 抵¥50，多出¥10不退 | 抵¥50（1次），第2张需另付¥50 |

## 积分批次类型

| 类型 | 标识 | 适用范围 |
|------|------|----------|
| 指定商品积分 | sku | 仅匹配指定SKU的商品 |
| 特殊商品积分 | special | 匹配特定标签的商品 |
| 品牌商品积分 | brand | 匹配特定品牌的商品 |
| 分类商品积分 | category | 匹配特定分类的商品 |
| 通兑商品积分 | general | 全场通用，电影票也可用 |

## 积分换算

1积分 = 1分 = ¥0.01（100积分 = ¥1.00）

## 选择页面返回机制

各选择页面通过 `localStorage` 的 `return_page` 字段动态返回对应的订单页面：
- 商品订单设置 `return_page = order_product.html`
- 电影票订单设置 `return_page = order_movie.html`

## localStorage 键值

| 键 | 用途 |
|----|------|
| order_type | 订单类型：product / movie |
| return_page | 选择页面返回地址 |
| nav_source | 来源标记：batch / coupon / movie |
| selected_batch_id | 选中的积分批次ID |
| selected_coupon_id | 选中的优惠券ID |
| selected_movie_card_id | 选中的电影次卡ID |
| order_products | 商品数据（JSON） |
